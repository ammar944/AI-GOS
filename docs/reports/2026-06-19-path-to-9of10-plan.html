⚑ Your question, answered straight

"Did we prove everything in CLI — is it now just a matter of wiring it to the agent?"  No. We proved ingredients in CLI, not the dish. "Just wiring" is the comforting story that has burned this project run after run ("fixed it" → live run disproves it). The honest gap to 9/10 is three different bodies of work, and only one of them is wiring:

① Wiring (real but small): inject the fact-ledger at dispatch, fix the getFacts() cross-section read bug, apply migrations.   ② Producer truth-fixes (the bulk — this is logic, not wiring): stop the sections that manufacture empty (BuyerICP) and manufacture confident (Paid Media), stop dropping promotable evidence (VoC), fix fan-out starvation.   ③ The proof loop (the keystone): there is no way to run a full deck end-to-end and grade it without a browser today — so none of ①/② is verifiable. Build that first or the cycle repeats.

Fix the manufacture-empty (BuyerICP), manufacture-confident (Paid Media's sibling logic), and ships-shallow (VoC) bugs. These three (best + worst + middle) calibrate the bar; each fix gated by the Phase-A harness on a real run, not a unit test or a solo dump.

Remaining producers + render honesty

D

The definition of done — 3-subject sweep

Full live runs for Ramp · Fathom · Plain, each through the gate, then an offline value-read judge. 8/10 first (clears the floor), then 9/10 (genuinely usable by a media buyer — no half-empty pages, no fabricated confidence).
This is the ONLY thing that counts as "it works." Not unit tests, not solo sections, not an average.
The one-line truth: the corpus and the ledger ingredients are proven; the producers that turn them into a deck still manufacture empty/confident output, and we have no way to prove a full deck end-to-end. 9/10 = build the proof loop → fix the producers against it → pass the 3-subject sweep. Reaching it is achievable systematically — it is not "just wiring," and it is not "simple patching."
