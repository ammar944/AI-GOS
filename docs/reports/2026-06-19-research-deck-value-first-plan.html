! The biggest finding: we throw away your own answers

Single biggest "50% empty" driver — and it's pure plumbing

The operator fills in ~15 fields about why customers buy, object, and churn — and not one section ever sees them.

The brief collects buying triggers, common objections, competitor frustrations, the situation before buying, desired transformation, value prop, guarantees, best client sources, the testimonial quote, sales process. Then corpusToResearchInput builds the agent's input from a closed key set that reads none of them, and the schema it must pass is .strict() so they'd be rejected anyway. So Voice-of-Customer, BuyerICP, and Demand go web-searching for pain points and objections the operator already typed in — and often come back emptier and more generic than the answer they were handed.

Same story with the URLs you supply: the case-study, pricing, and testimonial links never reach the scraper (it's switched on but never told the URL), so the highest-signal first-party pages go unfetched. And the final Paid Media plan never gets a readable summary of the five sections that ran before it — only a mechanical row-match — so it can't reason over the pains and quotes the pipeline just produced.

C The one wire we cannot get wrong

⚠ The fabrication door — where a fake segment could sneak back in

When BuyerICP grounds on a role/segment instead of a name, that role/segment token has to be re-checked against the live page. There are three places it could land, and only one is safe. The adversarial pass tried five attacks against the safe routing and all were blocked — the danger is purely a wiring slip.

STRICT — correct
Add the field to entityFieldNames. The full phrase must appear verbatim on the page (.every). Fabrication blocked.
SCHEMA-ONLY — worst
Add to the persona schema but forget the gate. Checked by nobody; the row even skips the re-probe. A fake segment ships. Worse than the old hole.
LENIENT — the trap
Add it to the prose-text list. The phrase fragments to single words; any one match passes (.some). The old fabrication hole, reopened.
VERIFIED: routing is fragile across two files by string-match only → make it ONE shared constant (compile-time safe) + one regression test that proves a fabricated segment is dropped and a real one is kept. Then the door stays shut.
