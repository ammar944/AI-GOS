02 / What we build with it

Two lanes. An offline ingestion lane lands raw transcripts and runs the one bounded LLM step behind a code-side anti-fabrication gate; a runtime lane re-derives the ranked cockpit deterministically on every page load. Signals are inputs, never re-extracted.

The anti-fabrication gate is code, not prompt. Every extracted signal quote must be a verbatim substring of a single transcript turn — normalized (NFKC → ASCII → lowercase → collapsed whitespace) and matched in the script, never trusted from the model. Quotes that appear only in the summary, span turns, or fall under 12 chars are rejected before write; speaker, date & quote_sha256 are resolved from the matched turn. The LLM proposes; deterministic code disposes.

03 / True-risk logic

Verbal signals set the tier — not churn

Each driver carries its own severity; the tier is the worst driver that fires, and the score is just a sort key beneath it. Crucially, the two fields that lie — raw churn_score and risk.incidents — are demoted out of the tier decision entirely.

22 clients sit at 3. A floor of 3 would flood 52% of the fleet into warning and destroy the radar; the field is unreliable by construction. It adds × 2 to the score for tie-breaking and shows as a de-emphasized, transparency-only column — it never sets a tier.

risk.incidentsdemoted · corroboration only

113 incidents across 26 clients, mostly false-positives from product copy. Admissible only alongside negative sentiment (a low-points warning) — never standalone, never critical. A transparency-only column, never a tier driver.

› Why churn_score can't set the tier

A floor of 5 (the shipped bug) never fires; a floor of 3 would flood 52% of the fleet into WARNING from churn alone, destroying the radar. The field is bimodal {0, 3} by construction — it carries almost no signal.

The genuine escalations (Zuppler / PathfinderCRO, both churn = 3) are promoted by verbal Fathom signals — not by churn. A tier-distribution sanity test asserts the pre-extraction critical set stays ≤ ~6, so the engine can never paint a half-red fleet.

05 / How it ships

Build sequence & the gates that prove it

Six steps, each verifiable. The deterministic pieces ship first; the one paid LLM step runs on a single client before the full sweep, honoring “paid APIs never loop without an abort condition.”

  1. MigrationTwo new tables + RLS internal-only SELECT + extend sl_refresh_runs run-kinds. Dated after the agency-intelligence migration so it applies later.
  2. Fathom ingestion (phase A)Deterministic raw sync → sl_fathom_transcripts. Attribution join; collisions fail the guard with zero writes.
  3. Anti-fabrication gatePure fathom/gate.ts — verbatim substring match. G1–G6 tests: verbatim kept, paraphrase / cross-turn / summary-only / len<12 rejected.
  4. Phase B extraction (sonnet)Only LLM step, Output.object + gate. First run --client=zuppler --extract-only, eyeball the quote, then the ~109-call sweep.
  5. True-risk engine + testsPure, no-LLM scoring. Fixtures F1–F13 green; Zuppler / PathfinderCRO / RapportScore must be CRITICAL.
  6. Cockpit UI + live verifyRisk-sorted overview + “Critical accounts” stat + per-client verbal-signals section. Authenticated browse: Zuppler & PathfinderCRO at top as CRITICAL, raw churn still 3.
