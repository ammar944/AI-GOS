# Research Document View — Visual Polish

**Date**: 2026-03-16
**Status**: Approved
**Branch**: `redesign/v2-command-center`

## Problem

The research document view (`/research/[sessionId]`) is functional but visually flat. All cards look identical, section headers are minimal, and the overall reading experience lacks the premium feel of the workspace. The view should feel like a polished deliverable — something you'd confidently share with a client or stakeholder.

## Approach

Data-Dense Report polish — upgrade card hierarchy and typographic rhythm within the existing dark theme and tab-based navigation. No structural changes, only visual refinements.

### Design System Context

- **Fonts**: Instrument Sans (headings), DM Sans (body), JetBrains Mono (mono/labels)
- **Theme**: Dark OLED with blue-tinted backgrounds (`--bg-base: rgb(7, 9, 14)`)
- **Accent**: `--accent-blue: rgb(54, 94, 255)` primary, section-specific accent colors exist
- **Cards**: `glass-surface` utility, `--bg-card`, `--border-subtle`
- **Existing section accent vars**: `--section-market`, `--section-competitor`, `--section-icp`, `--section-offer`, `--section-synthesis`, `--section-keyword`

## Design Sections

### 1. Document Header

**Current**: FileText icon + company name + date/stats row.
**Polish**:

- Remove the `FileText` icon — the heading provides sufficient context
- Company name: `font-heading text-xl font-semibold text-primary`
- Meta bar: date, section count, insight count separated by `·` middots in `text-xs font-mono text-tertiary`
- Gradient underline: 1px `bg-gradient-to-r from-[var(--accent-blue)]/20 to-transparent` spanning the content max-width, replaces the plain bottom gap

### 2. Section Headers (Document Mode)

**Current**: Module number text + label + description + hard `border-b`.
**Polish**:

- Module badge: render `Module 01` as a pill chip — `bg-[var(--accent-blue)]/10 text-[var(--accent-blue)] text-[10px] font-mono px-2 py-0.5 rounded-full inline-block`
- Section title: `font-heading text-lg font-semibold text-primary mt-1.5` (unchanged)
- Description: `text-sm text-tertiary leading-relaxed mt-1` (unchanged)
- Divider: replace `border-b border-white/[0.06]` with gradient line — `h-px bg-gradient-to-r from-[var(--accent-blue)]/20 to-transparent` at bottom of header block

### 3. Card Visual Hierarchy — Two Tiers

All cards currently share identical styling. Introduce two visual tiers to create scanning rhythm.

**Classification mechanism**: Hardcoded set in `CardRenderer`. No changes to `CardState` type.

```typescript
const HERO_CARD_TYPES = new Set(['stat-grid', 'strategy-card', 'competitor-card', 'pricing-card']);
// All other cardTypes are standard tier
```

**Hero cards** (`stat-grid`, `strategy-card`, `competitor-card`, `pricing-card`):
- Left border accent: `border-l-2 border-[var(--accent-blue)]/40`
- Increased padding: `p-6` (from `p-5`)
- Visible left border: 2px solid, testable
- Label with section-colored dot prefix

**Standard cards** (all other cardTypes):
- Keep `p-5` padding
- Subtler border: `border-white/[0.04]` (from `0.06`)
- No left accent

### 4. Card Labels

**Current**: `text-[10px] font-mono text-quaternary uppercase tracking-wider` — nearly invisible.
**Polish**:

- Size bump: `text-[11px]`
- Brightness bump: `text-tertiary` (from `text-quaternary`)
- Colored dot prefix: tiny 4px circle using `var(--accent-blue)` as a universal accent color (avoids needing section context in CardRenderer — keeps the component interface unchanged)

### 5. Navigation Bar — Document Mode Tweaks

**Current**: Pill-shaped tabs with background fill on active.
**Polish**:

- Active tab: bottom indicator line (2px `accent-blue`) instead of background fill — cleaner reading context
- Keep the rounded pill shape for inactive tabs
- ~~Card count badge~~ — removed (redundant with header stats, avoids data plumbing)

### 6. Print Refinements

**Current**: Basic `@media print` styles with `no-print` / `print-only` classes.
**Polish**:

- Page footer: "Generated by AIGOS" + date in `text-[10px] text-gray-400` centered at bottom
- Hero cards in print: left border `border-l-2 border-blue-400` to maintain visual hierarchy
- Ensure gradient dividers render as solid `border-gray-200` lines in print

- All workspace-mode components stay identical
- Card content components (`stat-grid.tsx`, `competitor-card.tsx`, etc.) are untouched — changes are in the wrapper/renderer layer only
- `CardState` type — no schema changes
- No new dependencies

## Success Criteria

1. Hero cards (`stat-grid`, `strategy-card`, `competitor-card`, `pricing-card`) have visible `border-l-2` and `p-6` padding; standard cards have `p-5` and no left border
2. Section headers render module number as pill badge with blue accent background
3. Card labels are `text-[11px] text-tertiary` with a 4px blue dot prefix — readable without straining
4. Active tab in document mode has 2px bottom indicator line instead of background fill
5. Print output: hero cards have left blue border, gradient dividers render as solid gray lines, footer reads "Generated by AIGOS"
6. No regressions in workspace mode — all changes gated behind `mode === 'document'`
