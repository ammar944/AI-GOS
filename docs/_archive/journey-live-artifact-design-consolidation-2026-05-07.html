1. Real UI patterns — observed, not invented

Every finding below was confirmed by a Refero screenshot or a design system doc. The "Apply to /journey" col is the bridge.

Manus.im — Wide Research feature page
Editorial serif headings + centered hero + muted warm neutrals (#F7F6F4 bg). Large rounded video thumbnail demos. Three-column comparison tables with highlighted "Manus" column.
→ The artifact surface should use editorial hierarchy (not raw markdown). Section decks, summary cards, evidence callouts. The report is a document, not a log.
Claude.ai — chat interface
Warm beige background (#F4F3EC). Chat messages appear as structured content blocks with document attachments shown in a right-side "Chat controls" panel. Typography uses Tiempos (serif headings) + Intercom/Styrene (UI). Streaming text with markdown rendering.
→ Right-side "Activity" rail should mirror Claude's Chat controls — collapsible, secondary, shows sources/files. Primary surface is the artifact.
Anthropic Workbench — console
Dark mode (#121212). Split two-panel comparison layout. Model/prompt dropdowns + version toggles. Prompts on left, responses on right. Footer action bar. "Show Prompts" toggle.
→ The artifact surface + activity rail split already matches this pattern. Keep it. Add a "Show activity" toggle like Workbench's "Show Prompts."
Cartesia Agents — dashboard
Dark-themed (#121212). Left vertical nav sidebar. "Your Agents" header with 3 creation cards (Text to Agent, Connect Your Code, Quickstart Template). Cards have icons + descriptions. "NEW" badge on sidebar items.
→ Section status badges: "writing," "complete," "queued" should be visible in the artifact. Use badge patterns from DESIGN.md (pill, mono 10px).
Cursor — design system (getdesign.md)
"AI-first code editor." Sleek dark interface. Gradient-based accents. Curated by VoltAgent team as a reference for "code editor UIs, developer tool marketing, and dark gradient designs."
→ Use gradient accents sparingly — only for the streaming cursor / active-writing indicator. DESIGN.md already bans decorative gradients. One exception: the active-writing pulse indicator.
Claude — design system (getdesign.md)
"Warm terracotta accent, clean editorial layout." For "AI chat interfaces, warm editorial layouts, and intellectual product pages." Both light and dark modes.
→ The artifact document (light-mode paper surface inside dark workspace) is the right direction. Contrast between dark UI chrome and light reading surface signals "this is the deliverable."
ChatGPT Deep Research — blog/product page
OpenAI Sans font. Research feature demo with sample query input mockup. Performance stats and charts. Content-heavy blog layout.
→ The agent's opening message should state intent + plan, like: "I'm building a source-backed GTM report. First, company context and evidence." Not "Deep Research Agent starting."
Parallel.ai — API key modal
Dark UI with white modal overlay. Mono font for code/keys. Clipboard copy icon. "CLOSE" button in uppercase. Clean, minimal dialogs.
→ Source citations should use mono font + copyable format. "Airtable newsroom, Jan 2026 [→]" — clickable, not just raw markdown links.

3. Component library — what the agent emits instead of Markdown

These components replace raw markdown dumps. Each maps to a content job identified in the gap report and confirmed by Reference app patterns.

EvidenceCard
A claim + source citation block. 2px left accent border (DESIGN.md callout pattern). Mono source line with clickable link. Used when the agent surfaces a substantiated claim.
Ref: Claude.ai structured content blocks, DESIGN.md callout blocks
SourceRail / SourceBadge
Compact source reference. Domain + date + favicon. Clickable. Appears inline next to claims or in the right rail. Replaces raw "[source]" markdown links.
Ref: Claude.ai "Chat controls" document panel, Parallel.ai API key display
SectionChapter
A named, numbered chapter block within the artifact. Has: title, status badge (writing/complete/queued), prose body, and child components (evidence cards, tables, callouts). Sections stack sequentially.
Ref: Manus.im editorial section layout, Anthropic Workbench split-panel
CompetitorMatrix
A comparison table following DESIGN.md table rules: mono headers, right-aligned numbers, tabular-nums, hover row highlight. Columns: name, strength, weakness, relevance score.
Ref: Manus.im 3-column comparison table, DESIGN.md table patterns
ExecutiveSummary
Top-of-artifact summary block. 2-3 sentence overview + key stats. Editorial hierarchy (larger type, more whitespace). Appears first, before sections.
Ref: ChatGPT Deep Research summary format, Manus.im hero sections
ICPBlock
Structured buyer profile: title, industry, size, pain points, buying triggers. Uses badge clusters for attributes. Not a prose paragraph — keyed fields.
Ref: Cartesia Agents card pattern (icon + label + description per attribute)
RecommendationStack
Numbered, prioritized recommendations. Each: title + rationale + confidence indicator (green/amber/red). Mono numbering. Copyable as a list.
Ref: Anthropic Workbench response panel structure, DESIGN.md list item patterns
ArtifactShell
The container that appears instantly on dispatch. Shows: company name, report title, section outline (numbered chips), active writing indicator, skeleton placeholders. The user sees the shape before content arrives.
Ref: Manus.im "shell first" loading pattern, Cursor progressive reveal

4. Motion choreography — the "feel" layer

The gap report's #6 flaw is "slow-feeling feedback." The UI must provide high-frequency meaningful changes. These patterns are derived from taste-skill rules (MOTION_INTENSITY=6, spring physics, no linear easing).

Phase 0 — Instant shell

ArtifactShell renders in <100ms. Title, company, outline chips, skeleton blocks. Zero animation on mount — content appears instantly (DESIGN.md rule).

artifact.start → shell mounted
artifact.outline.delta → outline chips appear
Phase 1 — Streaming content

Components stream in as deltas. Each new EvidenceCard or SectionChapter enters with a subtle opacity fade (150ms, ease-out). Writing cursor pulses (2s infinite, opacity 0.5→1→0.5).

artifact.component.delta →
  component mounts with fade-in
WritingIndicator pulses
Phase 2 — Section transitions

When a section completes, its status chip flips from "writing" (blue pulse) to "complete" (green solid, 200ms color transition). Next section chip shifts from "queued" to "writing." No layout shift — chips are pre-allocated.

section.complete →
  chip: pulse → solid green
  next chip: queued → active
Motion constraints (from DESIGN.md)
No card stagger entrance → content appears instantly No hover scale transforms → color/opacity only Easing: enter(ease-out) exit(ease-in) Duration: 100-200ms max Respect prefers-reduced-motion

6. Implementation sequence — what to build first

Wave 1 — This week
1 ArtifactShell component — instant mount with title, outline chips, skeleton blocks. Replaces the current "artifact appears when content arrives" behavior.
2 SectionChapter component — wraps each section with heading, status badge, prose body. Replaces raw markdown blocks.
3 Collapsible Activity rail — logs move to right rail with default-collapsed state. "Show activity" toggle. Reverses the current log-dominant layout.
4 Agent intent message — first message states plan + scope. "I'm building a source-backed GTM report..." Replaces generic "Agent starting" text.
Wave 2 — Next week
5 EvidenceCard component — structured claim + source block with 2px left accent. Replaces inline "[source]" markdown links.
6 CompetitorMatrix + ICPBlock — structured tables/badges for competitor and ICP data. Follows DESIGN.md table rules.
7 Section sequencing — backend continues but foreground shows "queued" until current chapter completes. Prevents visual interruption.
8 SourceRail — aggregated source list in activity rail. Domain + date + clickable links.
Wave 3 — Later (OpenUI-style generative UI)
9 Typed component event protocol — worker emits artifact.component.delta events with component type + props instead of raw markdown. Renderer maps to React components.
10 RecommendationStack + FollowUpComposer — prioritized recs with confidence indicators. User can request deeper dives on specific recs.
11 User steering — "sharpen ICP," "compare vs Notion," "turn this into outbound angles" edits the live artifact through the chat rail.

8. Verdict — can we hit Manus-grade?

Already right

The 3-panel shell (sidebar + workspace + chat rail) is correct. Dark chrome + light artifact document contrast is correct. The backend produces real content. The DESIGN.md tokens are sound and taste-skill validated.

The gap is presentation

Not capability — presentation. The artifact is being treated as a log entry instead of a premium writing surface. Fixing this is 80% front-end component work, 20% worker event format changes.

What "Manus-grade" means concretely

Instant shell → streaming structured components → sequential chapters → activity as trust layer → user steering. Every piece of this spec is buildable with existing stack components (shadcn/ui, Tailwind, Vercel AI SDK). No new infrastructure required.
