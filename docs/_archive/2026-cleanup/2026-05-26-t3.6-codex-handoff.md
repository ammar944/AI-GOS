## TASK 1 (primary) — completed-section render crash on `/research-v3`

**Symptom:** the moment any lab section flips to `complete`, `/research-v3` throws to the error boundary ("Something went wrong / Go to Dashboard").

**Root cause (CONFIRMED — code + DB):** the lab engine persists each artifact as an **envelope**. `research_artifact_sections.data` top-level keys are:
`id, body, runId, sources, verdict, createdAt, sectionId, confidence, sectionTitle, statusSummary`
— the **domain sub-sections live under `data.body`**. Confirmed for BuyerICP: `data.body` keys = `clusters, buyingContext, personaReality, icpExistenceCheck, awarenessDistribution`.

But `TypedArtifactRenderer` (`src/components/research-v2/typed-artifact-renderer.tsx:412-425`) dispatches by `zoneId` to the typed renderers in `src/components/research-v2/section-renderers/*.tsx`, which destructure the domain fields at the **top level**. E.g. `BuyerICPRenderer` (`section-renderers/buyer-icp.tsx:95-101`) does `const { icpExistenceCheck, personaReality, awarenessDistribution, buyingContext, clusters } = artifact;` then line 180 executes `awarenessDistribution.levels` → `awarenessDistribution` is `undefined` (it's in `artifact.body`) → **`TypeError: Cannot read properties of undefined`** → error boundary.

Path: `audit-state` route returns `sectionsByZone[zone] = { title, markdown, data }` (only for `status==='complete'`); `section-card.tsx` calls `pickPositioningTypedArtifact(body, zoneId)` which resolves to `body.data` (the envelope — it passes `isPositioningTypedArtifact` because `sectionTitle/verdict/statusSummary/confidence/sources` ARE at the envelope top level); that envelope is handed to the typed renderer, which can't find the domain fields nested under `.body`.

**Fix (render-layer, no data/persistence change):** in `pickPositioningTypedArtifact` (`src/types/positioning-artifact.ts`), after resolving the typed artifact, if it carries a `body` **object** (lab envelope), return a **flattened** artifact: the envelope meta (`sectionTitle, verdict, statusSummary, confidence, sources`) + `body`'s domain fields spread to the top level, and **drop envelope-only keys** (`id, runId, createdAt, sectionId, body`) so the generic `getSubSections` path doesn't render junk. Keep the existing source-filtering. This is a **no-op for already-flat artifacts** (managed-agents shape has no `body`), so it can't regress the `/research-v2` path.

**VERIFY a per-section field-name match (important):** the flatten fixes the *nesting*. But each lab section's `body` field names must match what its renderer destructures, or that section renders empty (no crash, but blank). For each of the 6, compare `data.body` keys (query below) against the renderer's destructure (`section-renderers/<section>.tsx`). If a section's lab schema uses different field names than the managed renderer expects, flag it (deeper issue — may need a per-section adapter or a renderer tweak). At minimum BuyerICP aligns.

Unit test in `src/types/__tests__/positioning-artifact.test.ts` (create or extend): envelope-shaped input (`{sectionTitle, verdict, statusSummary, confidence, sources, body:{...domain}, id, runId, createdAt, sectionId}`) → `pickPositioningTypedArtifact` returns domain fields at top level + meta preserved + envelope-only keys gone. And a flat (managed) input is returned unchanged.
