## Common pattern for every Section port

Each port produces the same 9 deliverables. The variations are sub-section names, Card types, cardinality minimums, and prior-art research terms.

### 1. Section schema file — `research-worker/src/agents/subagents/schemas/<section>.ts`

- Top-level `<Section>ArtifactSchema` with scalars: `sectionTitle`, `verdict`, `statusSummary`, `confidence: number (0-10)`, `sources: SourceSchema[]`
- Named sub-sections matching the canonical bullets in `docs/research-sections.md` — one per bullet, in the order shown there
- Each sub-section: `{prose: string (markdown), <cards>: HomogeneousCardArray}` — one Card type per array, NO discriminated unions
- Card schemas exported individually; use `.describe()` for every field
- NO `.min()/.max()` on arrays or numbers (Anthropic structured-output rejects these)
- Export `validate<Section>Minimums(artifact): {ok: boolean; errors: string[]}` — enforces cardinality minimums in TypeScript (the only place they can live)
- Import `SourceSchema` from `./_shared`

### 2. Schema tests — `research-worker/src/agents/subagents/schemas/__tests__/<section>.test.ts`

- Hand-crafted fixture with valid data for every field
- `MarketCategoryArtifactSchema.safeParse(fixture)` → success
- `safeParse` rejects on missing required sub-section fields
- `safeParse` rejects out-of-enum values
- `validate<Section>Minimums(fixture)` returns `{ok: true, errors: []}`
- `validate<Section>Minimums` fails when arrays go below the documented minimums
- `validate<Section>Minimums` fails when required-enum coverage is missing (e.g. competitor types missing one bucket)
- `validate<Section>Minimums` fails when confidence is outside 0-10

### 3. SKILL.md — `research-worker/platform-skills/<skill>/SKILL.md`

Full rewrite per the template in `docs/architecture/2026-05-14-positioning-audit-stack.md` § SKILL.md shape. Strip all legacy language:

- ❌ NO embedded Python validator (`Validator Source` section)
- ❌ NO references to `code_execution`, `/tmp/plan.json`, `validate.py`

### Card field guidance

### Cardinality minimums

- `offerMarketFit.proofPoints`: ≥3 with reported metric + source

- `redFlags.items`: ≥3 contradictions
- `sources`: ≥5

### IRON LAW essentials

- Self-data only: use the company's own reported numbers. External benchmarks ≠ the company's data.
- "Not disclosed" is a valid value and beats fabrication.
- A red flag requires a specific claimed-vs-actual contradiction, not a generic concern.
