Unified Onboarding Review — Layout Approach

All 39 fields in one scrollable page, grouped by category. Scraped values pre-filled, manual fields empty with placeholders. Required blockers marked.

A

Accordion Groups (Recommended)

Each category is a collapsible section. Groups with all fields filled show a green checkmark and collapse by default. Groups with missing required fields stay open. User can expand any group to review/edit.

Pros

  • Not overwhelming — user sees 7 group headers, not 39 fields at once
  • Auto-focuses attention on what needs filling
  • Groups with scraped data collapse out of the way

Cons

  • Extra click to expand a group to review pre-filled values
  • Slightly more component code
B

Flat Scrollable Sections

All 7 groups visible at once, separated by section headers. No collapsing. Sticky "Start Market Overview" button at the bottom. Missing required fields highlighted with amber borders.

Pros

  • Everything visible at a glance — no hidden state
  • Simpler implementation — just headers + fields
  • Easy to scan and edit anything

Cons

  • Long scroll — 39 fields is a lot of vertical space
  • Can feel form-like and tedious
C

Two-Column Card Grid

Each group is a card in a 2-column grid. Cards show field count and fill status (e.g., "5/7 filled"). Click a card to expand it inline and edit fields. Required cards have an amber dot.

Pros

  • Compact — all 7 groups visible without scrolling
  • Visual progress at a glance per group
  • Premium feel — card-based, not form-based

Cons

  • Two interactions to edit (click card → edit field)
  • Harder to review all values quickly
  • More complex expand/collapse behavior
