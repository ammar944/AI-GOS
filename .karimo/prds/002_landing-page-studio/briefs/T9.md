## Objective

Write `skills/landing-page-studio/scripts/regen-section.ts` — a tweak script that fully regenerates a named section of a landing page HTML file using Anthropic Claude Sonnet. Unlike `patch_text` (T8, Ollama, text-only), `regen_section` regenerates the entire section's HTML including layout, colors, and copy — while keeping all other sections untouched. It re-loads the BrandSpec and the chosen DirectionSpec from `output/<run>/directions.json` to maintain brand coherence.

---

## Context

**Parent Feature:** Landing Page Studio (PRD #002)

`regen_section` is the heavy tweak primitive. The user reaches for it when `patch_text` isn't enough — when they want to redesign a section's layout, not just reword it. Because it uses Anthropic Sonnet (same model as the original T5 generation), it can produce section HTML with the same quality as the original generation.

The script must re-load both BrandSpec and DirectionSpec so the regenerated section is coherent with the chosen direction (layout paradigm, color temperature, typographic register, OKLCH palette). It must not regenerate with generic defaults.

This task is part of **Wave 4** — T5, T6, and T7 must be complete before this is used. T8 and T9 are parallel tweak primitives — they do not depend on each other.

### Patterns to Follow

- **Per-skill Anthropic exemption** — Same pattern as T5. Use `anthropic(MODELS.CLAUDE_SONNET)` directly via `generateText`. Bypass `getGtmSkillLanguageModel()`.
- **`// TODO(skill-on-ollama)` comment required** — Per `ai-sdk-patterns.md`, any per-skill Anthropic fallback MUST include this comment. See exact format in Implementation Guidance.
- **Re-load DirectionSpec from `output/<run>/directions.json`** — Do not accept direction parameters as CLI flags (they could drift from the actual chosen direction). Read the file and use the direction matching `--direction <A|B|C>`.
- **Re-load BrandSpec from `output/<run>/brand-spec.json`** — Same reason — brand coherence requires the original spec.
- **`data-section` contract** — The regenerated section fragment MUST carry `data-section="<name>"` on its root element. This is a hard requirement for downstream T8 calls to work after a regen.
- **Same HTML constraints as T5** — OKLCH colors, no Lucide CDN, Tailwind via cdnjs only, no external scripts except Tailwind.

### Known Issues to Address

- **Model may return full page** — Even with explicit instructions, Sonnet occasionally wraps the section in a full HTML document. Detect `<!DOCTYPE` or `<html` in the response and error out rather than silently overwriting the file.
- **Section name must match exactly** — The `--section` value must match a `data-section` attribute in the current HTML. If it doesn't exist, error out with a list of available section names rather than silently doing nothing.
- **Color drift** — Regenerated section must use OKLCH colors from the DirectionSpec palette, not arbitrary new colors. Include the DirectionSpec's `primary_color_oklch` and `accent_color_oklch` in the prompt explicitly.

   - `--section <section-name>` — the `data-section` value identifying the section to regenerate

   - User message: serialized BrandSpec + DirectionSpec + current section HTML + user instruction
6. Validate the returned fragment:
   - Must not contain `<!DOCTYPE` or `<html`
   - Must contain `data-section="<section-name>"` on the root element
   - Must not contain hex colors in style attributes
   - Must not contain Lucide CDN script tags
7. Replace the target section in the full HTML with the regenerated fragment.
8. Write the mutated HTML back to the same file path.
9. Print: `Regenerated section "<section-name>" in direction-<id>.html`
10. Exit 0 on success, exit 1 on validation failure or model error.

### Section Extraction and Replacement

Reuse the same pattern from T8 (both use `node-html-parser`):

```typescript
import { parse } from "node-html-parser";

function extractSection(html: string, sectionName: string): string | null {
  const root = parse(html);
  const el = root.querySelector(`[data-section="${sectionName}"]`);
  return el ? el.outerHTML : null;
}

function replaceSection(html: string, sectionName: string, newHtml: string): string {
  const root = parse(html);
  const el = root.querySelector(`[data-section="${sectionName}"]`);
  if (!el) throw new Error(`Section "${sectionName}" not found`);
  el.replaceWith(parse(newHtml).firstChild!);
  return root.toString();
}

function listSections(html: string): string[] {
  const root = parse(html);
  return root.querySelectorAll("[data-section]").map(el => el.getAttribute("data-section") ?? "");
}
```

### Validation After Model Response

```typescript
function validateRegenFragment(fragment: string, sectionName: string): void {
  if (fragment.includes("<!DOCTYPE") || fragment.toLowerCase().includes("<html")) {
    throw new Error("Model returned full HTML page instead of section fragment.");
  }
  if (!fragment.includes(`data-section="${sectionName}"`)) {
    throw new Error(`Model stripped data-section="${sectionName}". Section contract broken.`);
  }
  if (/style="[^"]*#[0-9a-fA-F]{3,8}/i.test(fragment)) {
    throw new Error("Model introduced hex colors in style attributes.");
  }
  if (/src=["'][^"']*lucide/i.test(fragment)) {
    throw new Error("Model included Lucide CDN script. Must use inline SVG.");
  }
}
```

### Edge Cases

- If `--section` doesn't match any `data-section` in the file: exit 1 with "Section '<name>' not found. Available: <comma-separated list from listSections()>"
- If `output/<run>/directions.json` doesn't have direction `<id>`: exit 1 with "Direction <id> not found in directions.json"
- If `maxOutputTokens` is hit and fragment is truncated (model output ends mid-tag): detect unclosed tags and exit 1
- If `--instruction` is omitted: use the default prompt text ("Improve this section...") — not an error
- After successful regen, prompt user to re-run T7 QA and optionally T8 for copy tweaks
