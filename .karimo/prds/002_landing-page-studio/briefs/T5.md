## Objective

Write `skills/landing-page/scripts/generate-html.ts` that takes one `DirectionSpec` from `directions.json` and generates a complete, self-contained HTML landing page using Anthropic Claude Sonnet (per-skill exemption). The script runs 3 times in parallel (once per direction A/B/C), each producing a distinct HTML file. The prompt is built by extending — not importing — the existing `skills/landing-page/prompts/system.md`.

### Patterns to Follow

- **`generateText` not `generateObject`** — HTML output is free-form text, not structured JSON. Use `generateText` from `"ai"`.
- **`// TODO(skill-on-ollama)` comment required** — Per `ai-sdk-patterns.md`, any per-skill Anthropic fallback MUST include this comment naming the failure mode. See exact format below.
- **Copy, do not import, the system prompt** — `skills/landing-page/prompts/system.md` is the source. Copy its content into `prompts/html-generator.md` and extend it with direction-specific injection. Never `fs.readFileSync` the original at runtime — the brief says "copy + extend, do not import."
- **Lucide icons must be inlined** — No CDN `<script>` tag for Lucide. Inline the SVG paths directly in the HTML. Tailwind via cdnjs is acceptable.

### Known Issues to Address

- **Two diverged system prompts exist** — `skills/landing-page/prompts/system.md` has CLI vs API variants. Read both before writing the extended prompt to understand what differs. The API variant is the correct base.
- **Huashu, Impeccable, Taste, UIUXProMax are phantom modules** — These are prompt text sections, not real imports. Do not attempt to import them.
- **`--style=VALUE` arg form silently corrupts the prompt** — If the CLI uses `--style=VALUE` syntax, it corrupts the prompt string. Use `--style VALUE` (space-separated) in any CLI arg parsing.

### CSP String

> ⚠️ DISCREPANCY found 2026-05-01 (post-write) — the two locked sources disagree:
>
> **scope-decisions.md item 4**: `default-src 'self' data: blob:; script-src 'unsafe-inline'; style-src 'unsafe-inline' 'self'; img-src 'self' data: https:; connect-src 'none'; frame-ancestors 'self'`
>
> **tasks.yaml T6 acceptance**: `default-src 'self' data; script-src cdnjs.cloudflare.com 'unsafe-inline'; style-src cdnjs.cloudflare.com 'unsafe-inline' 'self'; img-src 'self' data https; connect-src 'none'; frame-ancestors 'self'`
>
> Differences: (a) tasks.yaml drops `:` after `data` and `https` (technically invalid CSP — schemes need colons), (b) tasks.yaml adds `cdnjs.cloudflare.com` to script-src and style-src (required to load Tailwind/lucide from cdnjs), (c) scope-decisions.md adds `blob:` to default-src (tasks.yaml drops it).
>
> The CORRECT merge (recommended for implementation): `default-src 'self' data:; script-src cdnjs.cloudflare.com 'unsafe-inline'; style-src cdnjs.cloudflare.com 'unsafe-inline' 'self'; img-src 'self' data: https:; connect-src 'none'; frame-ancestors 'self'` — keeps colons (correct CSP), adds cdnjs (needed for Tailwind), drops `blob:` (no blob URLs in single-file HTML output). T5 and T6 must use the SAME string. Reviewer should confirm or override before T6 implements.

### data-section Attributes

Every structural section must carry this attribute for T9 (regen_section) to work correctly:

```html
<section data-section="hero">...</section>
<section data-section="features">...</section>
<section data-section="social-proof">...</section>
<section data-section="cta">...</section>
<footer data-section="footer">...</footer>
```

The exact section names don't need to match this list, but must be consistent and machine-readable. T9 depends on these for surgical section replacement.

### Lucide Inline SVG Pattern

Do not use:
```html
<!-- WRONG: CDN script -->
<script src="https://cdn.jsdelivr.net/npm/lucide@latest/dist/umd/lucide.min.js"></script>
```

Use instead:
```html
<!-- CORRECT: inline SVG -->
<svg xmlns="http://www.w3.org/2000/svg" width="24" height="24" viewBox="0 0 24 24"
     fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round"
     stroke-linejoin="round">
  <!-- paste path data from lucide source for the specific icon -->
  <polyline points="20 6 9 12 4 8 4 16"/>
</svg>
```

The prompt must instruct the model to inline SVG paths. Include 2-3 example icon SVGs in the prompt as references.
