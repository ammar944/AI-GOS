## Objective

Write `skills/landing-page/scripts/post-process.ts` — a deterministic, zero-AI post-processing script that runs on each generated HTML file after T5. ## Research Context

### Patterns to Follow

- **Pure TypeScript, no AI SDK** — This script makes no model calls. No import from `"ai"`, no provider calls.
- **DOM parsing via `node-html-parser` or regex** — For CSP injection and `data-section` preservation, prefer a lightweight HTML parser over regex. `node-html-parser` is the recommended choice (check `skills/landing-page/package.json` for availability; if absent, add it).
- **In-place file mutation** — Read the HTML file, transform, write back to the same path. Do not create new files.
- **Preserve `data-section` attributes** — Any DOM transformation must not strip `data-section` attributes. T9 depends on them.

### Known Issues to Address

- **OKLCH normalization is lossy** — Converting hex/rgb to OKLCH requires a color space library. Use `culori` (npm) for accurate conversion. Do not hand-roll the conversion math.
- **Lucide CDN detection** — Match both `cdn.jsdelivr.net/npm/lucide` and `unpkg.com/lucide` patterns. The model may use either CDN.
- **CSP idempotency** — If T5 already injected the CSP meta tag, do not inject a second one. Check for existing `<meta http-equiv="Content-Security-Policy"` before inserting.

## Implementation Guidance

### CSP Injection Pattern

> ⚠️ DISCREPANCY between scope-decisions.md item 4 and tasks.yaml T6 acceptance (see T5 brief for full diff). RECOMMENDED string for both T5 and T6 (must match):
>
> ```typescript
> const CANONICAL_CSP =
>   "default-src 'self' data:; " +
>   "script-src cdnjs.cloudflare.com 'unsafe-inline'; " +
>   "style-src cdnjs.cloudflare.com 'unsafe-inline' 'self'; " +
>   "img-src 'self' data: https:; " +
>   "connect-src 'none'; " +
>   "frame-ancestors 'self'";
> ```
>
> Justification: keeps colons after `data`/`https` (tasks.yaml dropped them — invalid CSP), adds `cdnjs.cloudflare.com` (required to load Tailwind + lucide from cdnjs), drops `blob:` (no blob URLs in single-file HTML output). Reviewer should confirm before implementation. T5 brief has the same text — keep them synchronized if changed.

```typescript
import { parse } from "node-html-parser";

function injectCSP(html: string): { html: string; injected: boolean } {
  const root = parse(html);
  const head = root.querySelector("head");
  if (!head) return { html, injected: false };

  const existing = head.querySelector('meta[http-equiv="Content-Security-Policy"]');
  if (existing) return { html, injected: false };

  const cspTag = `<meta http-equiv="Content-Security-Policy" content="${CANONICAL_CSP}">`;
  head.insertAdjacentHTML("afterbegin", cspTag);
  return { html: root.toString(), injected: true };
}
```

### OKLCH Normalization Pattern

```typescript
import { parse as parseColor, formatOklch, toOklch } from "culori";

function normalizeColorsInStyle(styleValue: string): { value: string; count: number } {
  let count = 0;
  // Match hex, rgb(), hsl(), named colors in style attribute values
  const normalized = styleValue.replace(
    /(#[0-9a-fA-F]{3,8}|rgb\([^)]+\)|hsl\([^)]+\)|(?<!\w)(red|blue|green|black|white|gray|grey)(?!\w))/g,
    (match) => {
      const color = parseColor(match);
      if (!color) return match;
      const oklch = toOklch(color);
      if (!oklch) return match;
      count++;
      return `oklch(${oklch.l.toFixed(4)} ${oklch.c.toFixed(4)} ${(oklch.h ?? 0).toFixed(2)})`;
    }
  );
  return { value: normalized, count };
}
```

### Lucide Removal Pattern

```typescript
function removeLucidenCDN(html: string): { html: string; count: number } {
  let count = 0;
  const cleaned = html.replace(
    /<script[^>]+src=["'][^"']*lucide[^"']*["'][^>]*><\/script>/gi,
    () => { count++; return ""; }
  );
  return { html: cleaned, count };
}
```

### Edge Cases

- If a direction HTML file doesn't exist (e.g., T5 failed for one direction): log "Skipping direction-X.html — file not found" and continue with the others.
- If `culori` can't parse a color string: leave it unchanged, log "Could not normalize color: <value>" at debug level.
- If `node-html-parser` fails to parse the HTML (severely malformed): catch the error, log it, and write the original file back unchanged.
- Named colors like `currentColor`, `transparent`, `inherit` — skip these, they are not actual color values.
