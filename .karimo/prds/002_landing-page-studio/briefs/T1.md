## Context

AIGOS uses a **skill-first architecture** (v3). Every pipeline stage is a self-contained skill folder. The reference implementation is `skills/research-competitor/` — this task replicates its folder structure for the new `landing-page-studio` skill.

The canonical per-skill layout (from CLAUDE.md, locked decision):

```
skills/landing-page-studio/
  SKILL.md          # Human-readable skill doc (purpose, inputs, outputs, usage)
  skill.json        # Machine-readable manifest (id, version, entry, schema refs)
  package.json      # Skill-scoped deps — no imports from outside this folder
  tsconfig.json     # Skill-scoped TS config
  scripts/          # Orchestrator + step scripts (implementations go here in later tasks)
  assets/           # Static assets, prompt templates, vocab files
  references/       # External reference material, research docs
  example/          # Example inputs/outputs for testing and docs
```

**Critical rule:** Skills must be self-contained. No imports from `src/lib/` or any path outside `skills/landing-page-studio/`. This skill will eventually be published as a standalone GitHub repo.

The landing-page-studio skill takes a `BrandSpec` input and produces a styled landing page. Wave 1 only scaffolds the structure — no implementation logic yet.
