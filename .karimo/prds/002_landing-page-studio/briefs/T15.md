## Gotchas

1. **Banner-only edit** — the entire scope is one insertion. If you find yourself modifying anything else, stop. The task is done when the banner is inserted.
2. **Idempotency** — check for an existing banner before inserting. If `grep -q 'deprecated' skills/landing-page/README.md` returns 0, the banner is already there. Exit without changes.
3. **Relative path correctness** — `../landing-page-studio/` works because both skill folders are siblings under `skills/`. Verify with: `ls skills/landing-page-studio/README.md` (must exist, i.e., T14 must be complete before T15 runs).
4. **GFM alert syntax** — `[!WARNING]` only renders as a callout on GitHub. In plain markdown viewers it renders as a blockquote, which is acceptable. Do not use HTML `<div class="warning">` — keep it in standard Markdown.
5. **Wave 7 dependency** — this task must run AFTER T10 and T14 are complete so that `../landing-page-studio/README.md` actually exists when the banner links to it. If the target doesn't exist yet, the link will be broken. Verify: `ls skills/landing-page-studio/README.md` before inserting.
