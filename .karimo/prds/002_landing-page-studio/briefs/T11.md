## Context

### What a Slash Command Is (vs SKILL.md)

These are two different loader mechanisms:

| Mechanism | File | Who Uses It | How Loaded |
|-----------|------|-------------|------------|
| Slash command | `.claude/commands/<name>.md` | Claude Code (interactive) | User types `/<name>` in Claude Code session |
| SKILL.md | `skills/<name>/SKILL.md` | Any Claude agent (Claude Code, API, etc.) | Agent reads SKILL.md to understand the skill |

A slash command is **Claude Code-specific**. It is a markdown prompt template that Claude Code surfaces as a `/` command. When the user types `/landing-page-studio`, Claude Code reads this file and uses it as the system context for that interaction.

A SKILL.md (T12) is a generic capability description any Claude agent can read to understand what the skill does and how to invoke it. They serve different audiences.

### Canonical Pattern

The reference slash command is `.claude/commands/research-competitor.md`. Its structure:

1. YAML frontmatter with `description` field (shown in Claude Code's command picker)
2. A brief purpose statement
3. Argument specification (what the user should provide)
4. What the command does step by step
5. Delegation to the CLI: "Run `npx tsx skills/<name>/scripts/cli.ts ...`"

### Three-Layer Architecture (from OMC observations)

AI-GOS slash commands follow a 3-layer architecture:
1. **Command file** (`.claude/commands/*.md`) — the user-facing prompt template
2. **Skill script** (`skills/<name>/scripts/cli.ts`) — the execution layer
3. **Skill package** (`skills/<name>/`) — the self-contained skill folder

The slash command is layer 1. It should be thin — its job is to collect user input and call layer 2.

```markdown
# Landing Page Studio

Generate a complete SaaS landing page (3 HTML variants, one per direction) from a BrandSpec JSON file.

## Usage

Provide a path to your BrandSpec JSON file:

```
/landing-page-studio path/to/brand-spec.json
```

Or with an output directory:

```
/landing-page-studio path/to/brand-spec.json --out path/to/output/
```

## What This Does

1. Validates your BrandSpec against the schema (T4)
2. Picks N directions from the direction catalogue (T5)
3. Plans sections for each direction (T6)
4. Renders 3 HTML files, one per direction (T7)

## Execution

Run the full pipeline:

```bash
npx tsx skills/landing-page-studio/scripts/cli.ts full \
  --brand $BRAND_SPEC_PATH \
  --out ${OUTPUT_DIR:-out/}
```

Where `$BRAND_SPEC_PATH` is the path you provided above.
```

### User Input Handling

The slash command receives the user's input as `$ARGUMENTS` (Claude Code convention). The command body should reference `$ARGUMENTS` to pass the brand spec path to the CLI.

## Gotchas

1. **Claude Code command discovery** — Claude Code discovers slash commands by scanning `.claude/commands/*.md`. The file must be in exactly that directory with a `.md` extension.
2. **`$ARGUMENTS` placeholder** — Claude Code substitutes `$ARGUMENTS` with whatever the user typed after the command name. Use this to pass the brand spec path.
3. **Do not add `allowed-tools` or other frontmatter fields** unless you've confirmed the research-competitor command uses them — unrecognized frontmatter may cause Claude Code to ignore the file.
