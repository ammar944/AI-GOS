## The 12 Gates (read from source — see TODO above)

The implementer must open `src/lib/ai/landing-page/quality-gates.ts` and identify all 12 gate definitions. Typical QA gates for landing page HTML validation include checks like:

- **Minimum word count** — page has sufficient copy (not a stub)
- **CTA presence** — at least one call-to-action element exists
- **Heading hierarchy** — H1 exists, H2s present, no skipped levels
- **No placeholder text** — no `[PLACEHOLDER]`, `Lorem ipsum`, `TODO` strings in output
- **No broken links** — href values are not empty or `#`
- **Meta description** — `<meta name="description">` tag present
- **Image alt text** — all `<img>` tags have non-empty `alt` attributes
- **Contact/social proof** — at least one trust signal element
- **Mobile viewport** — `<meta name="viewport">` tag present
- **No inline scripts** — no `<script>` tags in body (security gate)
- **Brand name present** — `brandName` appears in the HTML content
- **Color contrast hint** — checks for explicit color declarations (not a full WCAG check)

**These are illustrative only.** Read the actual source to get the real 12 gates.
