## Objective

Write `skills/landing-page-studio/scripts/patch-text.ts` — a tweak script that accepts a user instruction (e.g., "make the headline more aggressive") and a target section identifier, then uses Ollama via `getGtmSkillLanguageModel()` to rewrite only the text content within that section. The HTML structure, CSS classes, OKLCH colors, and `data-section` attributes must be preserved. Because it uses Ollama, this tweak is free (no Anthropic cost).

## Research Context

### Patterns to Follow

- **`getGtmSkillLanguageModel()`** — Ollama chokepoint. Import from `src/lib/gtm/skill-model.ts`. Same pattern as T4. No Anthropic import here.
- **`generateText` not `generateObject`** — Text patch output is a string (the rewritten section's inner HTML). Use `generateText`.
- **`data-section` attribute as selector** — Use the `data-section` attribute value to locate the target section in the HTML. This is the contract established by T5 and preserved by T6.
- **Preserve HTML structure** — The model must be instructed to return ONLY the inner HTML of the target section, not a full page. The script wraps it back into the original document.
- **12 QA gates** — The patched HTML must pass the same T7 QA checks. Run a lightweight in-process check after patching, or instruct the user to run T7 after patching.

### Known Issues to Address

- **Ollama may rewrite structure** — The prompt must explicitly say "return only the text content changes, do not alter HTML tags, CSS classes, or inline styles." If the model returns a full HTML document instead of just the section content, detect this and error out rather than silently overwriting.
- **OKLCH colors must survive** — The model must not convert OKLCH colors to hex/rgb. Include an explicit constraint in the prompt.
- **`data-section` must survive** — If the model strips the `data-section` attribute from the returned HTML fragment, T9 breaks. Check for this before writing.

**File Dependencies:**
- Input/Output: `output/<run>/html/direction-<id>.html` (mutated in-place)

## Requirements

1. Accept CLI flags:
   - `--run <run-id>`
   - `--direction <A|B|C>` — which HTML file to patch
   - `--section <section-name>` — the `data-section` value identifying the target section
   - `--instruction <text>` — user's natural-language tweak instruction (quoted string)
2. Read `output/<run-id>/html/direction-<id>.html`.
3. Extract the HTML element with `data-section="<section-name>"` (including its inner content).
4. Call `getGtmSkillLanguageModel()` then `generateText` with:
   - A system prompt instructing the model to rewrite only text nodes, preserve all HTML tags, CSS classes, inline styles, OKLCH colors, and `data-section` attributes
   - A user message containing the extracted section HTML + the user's instruction
5. Validate the returned fragment:
   - Must not contain `<!DOCTYPE` or `<html` (model returned full page → error)
   - Must preserve the `data-section` attribute on the root element
   - Must not contain hex colors (`#[0-9a-fA-F]`) in style attributes
6. Replace the original section in the full HTML with the rewritten fragment.
7. Write the mutated HTML back to the same file path.
8. Print: `Patched section "<section-name>" in direction-<id>.html`
9. Exit 0 on success, exit 1 on validation failure.

### Section Extraction Pattern

```typescript
import { parse } from "node-html-parser"; // reuse dep from T6

function extractSection(html: string, sectionName: string): string | null {
  const root = parse(html);
  const section = root.querySelector(`[data-section="${sectionName}"]`);
  return section ? section.outerHTML : null;
}

function replaceSection(html: string, sectionName: string, newHtml: string): string {
  const root = parse(html);
  const section = root.querySelector(`[data-section="${sectionName}"]`);
  if (!section) throw new Error(`Section "${sectionName}" not found`);
  section.replaceWith(parse(newHtml).firstChild!);
  return root.toString();
}
```

### Validation After Model Response

```typescript
function validateFragment(fragment: string, sectionName: string): void {
  if (fragment.includes("<!DOCTYPE") || fragment.toLowerCase().includes("<html")) {
    throw new Error("Model returned full HTML page instead of section fragment. Aborting patch.");
  }
  if (!fragment.includes(`data-section="${sectionName}"`)) {
    throw new Error(`Model stripped data-section="${sectionName}" attribute. Aborting patch.`);
  }
  const hexInStyle = /style="[^"]*#[0-9a-fA-F]{3,8}[^"]*"/i.test(fragment);
  if (hexInStyle) {
    throw new Error("Model introduced hex colors in style attributes. Aborting patch.");
  }
}
```
