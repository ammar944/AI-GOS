## Objective

Write a script `skills/landing-page/scripts/plan-directions.ts` that reads a validated BrandSpec (produced by T3) and calls the Ollama model via `getGtmSkillLanguageModel()` to produce exactly 3 disjoint visual direction objects. Each direction must be meaningfully distinct — different layout paradigm, color temperature, or typographic register. Output is written to `output/<run>/directions.json`.

---

## Context

**Parent Feature:** Landing Page Studio (PRD #002)

The landing-page-studio pipeline generates 3 divergent HTML landing pages from a single BrandSpec, then lets the user pick one and tweak it. The Direction Planner (T4) is the bridge between validated brand input (T3) and HTML generation (T5). It runs once per generation request and its output drives 3 parallel T5 calls.

"Disjoint" means the 3 directions must not be cosmetically similar. The model must be prompted to maximize visual distance between picks — e.g., one minimal/editorial, one bold/hero-image-driven, one card/grid-based.

This task is part of **Wave 3** — all of T1 (BrandSpec schema), T2 (CLI entrypoint), and T3 (BrandSpec validator) must be complete before this runs.

### Prompt Engineering

The system prompt must explicitly instruct the model to maximize visual distance:

```
You are a visual direction planner for landing pages. Given a BrandSpec, 
produce exactly 3 landing page directions that are maximally distinct.
Rules:
- No two directions may use the same layout_paradigm AND color_temperature combination
- No two directions may use primary colors from the same hue family (within 60 degrees on the color wheel)
- All colors must be expressed as OKLCH strings: oklch(L C H) where L is 0-1, C is 0-0.4, H is 0-360
- Each direction must feel like a different design studio made it
```

### Disjointness Refinement in Zod

```typescript
// schemas/directions.ts
import { z } from "zod"; // Zod v3

export const DirectionSpecSchema = z.object({
  id: z.enum(["A", "B", "C"]),
  label: z.string(),
  layout_paradigm: z.enum(["hero", "editorial", "grid", "split", "card-stack"]),
  color_temperature: z.enum(["warm", "cool", "neutral"]),
  typographic_register: z.enum(["serif-led", "sans-led", "mono-accent"]),
  primary_color_oklch: z.string().regex(/^oklch\(/),
  accent_color_oklch: z.string().regex(/^oklch\(/),
  rationale: z.string(),
});

export const DirectionPlanSchema = z.object({
  directions: z.array(DirectionSpecSchema).length(3).refine(
    (dirs) => {
      const combos = dirs.map(d => `${d.layout_paradigm}:${d.color_temperature}`);
      return new Set(combos).size === 3;
    },
    { message: "Directions must not share layout_paradigm+color_temperature combination" }
  ),
});
```

### Retry Pattern

```typescript
async function planWithRetry(brandSpec: BrandSpec, maxAttempts = 2) {
  for (let attempt = 1; attempt <= maxAttempts; attempt++) {
    try {
      const { object } = await generateObject({ model, schema: DirectionPlanSchema, prompt: buildPrompt(brandSpec) });
      return object;
    } catch (err) {
      if (attempt === maxAttempts) throw err;
      console.error(`Attempt ${attempt} failed, retrying...`, err);
    }
  }
}
```
