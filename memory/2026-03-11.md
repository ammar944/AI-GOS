- Added a dev-only Journey research sandbox at `/test/journey-research` that reuses the real async research pipeline. It dispatches the existing worker/runners through `dispatchResearchForUser`, writes into a sandbox-scoped `journey_sessions.user_id`, and surfaces status/results through the existing realtime + job activity hooks without touching the live Journey row.
- The sandbox API lives at `/api/journey/dev/research-sandbox` and supports seeding from the live Journey session, loading generated context drafts, running one section at a time, and clearing either one section or the full sandbox safely.
- Follow-up pass tightened sandbox reliability: dispatch failures now return an HTTP error instead of a false-success payload, sandbox context drafts merge sandbox artifacts over live artifacts for missing dependencies, and legacy/canonical section aliases are normalized so seeded sessions stay readable even if old keys exist in `research_results`.
- The sandbox UI now shows backend readiness/capabilities, current backend sequence, dependency warnings for the selected section, and inline previews for Strategic Synthesis, Keywords, and Media Plan using the real persisted shapes. Media Plan preview is powered by new subsection-card support rather than a separate mock renderer.
