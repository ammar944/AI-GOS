# Peter Steinberg

Engineer Karpathy references as the canonical example of multi-agent orchestration. Called out for innovating in five dimensions of claw design simultaneously.

## Why he's in this wiki

Karpathy's aspirational reference for what "correct use of multiple coding agents" looks like. "Ideally you want to become like Peter Steinberg — multiple Codex agents on one monitor, 10 repos checked out, each 20-minute task dispatched in parallel."

## The five dimensions of claw innovation

Per Karpathy:

1. **Persona / soul document.** The claw's identity, baseline values, how it talks.
2. **Memory system.** More sophisticated than default agents.
3. **Fun.** The character can surprise.
4. **WhatsApp portal.** Communication channel.
5. **Personality calibration.** Tuning emotional response.

## Why this matters for AIGOS

The Steinberg pattern is what AIGOS's `/team` orchestration is reaching for. The research-worker pipeline already does something similar (parallel runner dispatch), but the "10 repos, 20-minute tasks" pattern is for developer work, not end-user-facing product.

## What we don't know (yet)

No primary source from Steinberg himself in this wiki. Everything here is Karpathy's recollection. If a direct source appears — talk, blog, GitHub readme — ingest it and enrich this page.
